Phase 3 — Three-file data architecture
File 1 — gatekeeper
AISHE_CODE/U-1284.xlsx
4 columns only: StudentID, AISHE_Code, student_password, College_name. Used purely to verify identity at login. Lightweight and fast.
File 2 — full data
password_generated_files/ password_generated_*.xlsx
All 44 columns — scores, attendance, rank, department, activities, sports, ML model data, feedback. Loaded once after identity is confirmed.
File 3 — LLM context
student_chat_contexts/ U-1284_S0009.txt
Generated once per login from File 2. Every chatbot message reads from this .txt as LLM context — no re-reading Excel on every question.
