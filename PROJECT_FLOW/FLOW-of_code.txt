Three files are used at different stages:
Step 1 — AISHE_CODE/U-1284.xlsx (4 columns only: StudentID, AISHE_Code, student_password, College_name)
Used purely to verify the student exists for that university. It's lightweight and fast.
Step 2 — password_generated_files/password_generated_*.xlsx (all 44 columns — full student data)
Once identity is confirmed, the code scans this folder to pull the full row: scores, attendance, department, rank, feedback, sports, activities — everything.
Step 3 — student_chat_contexts/U-1284_S0009.txt (generated once per login)
The full student row is written to this text file. Every chat message reads from this .txt and sends it to the LLM as context. This avoids re-reading Excel on every question.
So the chatbot answers are based on the 44-column password_generated_*.xlsx data — not the AISHE file. The AISHE file is only a gatekeeper.
